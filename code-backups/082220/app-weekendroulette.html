Weekend Roulette

Description

Weekend Roulette is app that allows users to leave the weekend decision making to the app. The app provides two day activities and two evening ones with the 'spin of a wheel' (click of a button). Give it a spin!

Process

The app makes calls to four APIs - a hiking trail API, biking trail API, latitude/longitude API, and a restaurant API - in order to obtain the activity selections for users. While using the app, users experience some simple CSS animations that add to the enjoyment and experience of the app.

The free APIs being used all had some constraints to work with and around. All three activity APIs required latitude and longitude coordinates to be provided as a parameter which required adding the additional API call. I decided to have UIs with separate API calls - one that would return the coordinates and another that would return the results. Although, this added another form for users to submit, it simplified the API calls and made it easier to determine if there was an error in user input or with the requests.

The restaurant API could return hundreds of results for a location, but it would only return 20 at a time and the range had to be specified in the request (i.e. 1-19 or 340-359). This posed a problem since ideally the activities given to a user would be two random selections among all the results possible - not just the first 20. I solved this by generating random numbers to enter as the parameter range before making the API call.

Another problem that needed to be solved was the sequence of fetching the data to return random activities depending on whether a user wanted all three activities choices (biking, hiking, and restaurants) or a combination of two, or just one.

Future Versions

Future versions of the app will add more APIs, such as "points of interest" from Google Maps and music from Bands in Town, to add more types of activities. In addition, future versions will include an improved restaurant API and using Google Geocaching for latitude and longitude.
